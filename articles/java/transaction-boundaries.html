Spring 事务边界不要只看注解

从代理调用、异常类型和外部系统交互理解事务真正生效的范围。

2026-03-19

  2. 回滚规则需要明确

事务依赖代理调用

@Transactional 通常由 Spring 代理实现。同一个类内部直接调用事务方法， 可能绕过代理，导致事务没有按预期开启。

回滚规则需要明确

默认情况下，运行时异常会触发回滚，受检异常不一定回滚。业务代码不应 依赖模糊的默认行为，可以通过 rollbackFor 明确表达事务约束。

数据库事务管不到外部系统

消息发送、HTTP 调用和文件写入不属于数据库事务。需要跨系统一致性时， 应考虑本地消息表、事务消息、幂等消费或补偿任务，而不是扩大方法范围。
