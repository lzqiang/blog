# 个人静态博客设计

## 目标

构建一个以 Markdown 为内容源的个人博客。博客从根目录的 `index.html`
进入，生成结果无需启动服务器，可通过双击文件直接在浏览器中浏览。
Markdown 只用于写作，访客看到的是排版后的网页，而不是 Markdown 原文。

## 内容范围

首版提供四个固定分类：

- Java
- 人工智能
- 文学创作
- 个人随笔

首页展示全部分类入口和按日期倒序排列的最新文章。首版不包含搜索、
评论、后台管理、在线编辑器或其他需要服务端支持的功能。

## 技术方案

使用 Node.js 编写轻量静态站点生成器。作者只维护 `content/` 中的
Markdown 文件；运行 `npm run build` 后，生成器输出完整 HTML 页面。

所有页面资源和链接使用相对路径，确保从 `file://` 协议打开时仍能正常
访问首页、分类页、文章页、样式、脚本和图片。

计划目录：

```text
blog/
├── index.html
├── articles/
├── categories/
├── content/
│   ├── java/
│   ├── ai/
│   ├── literature/
│   └── essays/
├── assets/
├── templates/
├── scripts/
│   └── build.js
├── tests/
└── package.json
```

其中 `content/`、`templates/` 和 `scripts/` 是内容及生成逻辑；
`index.html`、`articles/`、`categories/` 和浏览所需的 `assets/`
是构建结果。

## 文章格式

每篇文章是一个 Markdown 文件，并在开头声明元数据：

```markdown
---
title: Java 并发编程笔记
date: 2026-06-10
category: Java
summary: 梳理线程、锁和并发工具的核心概念
---

# 正文标题

这里开始写文章……
```

必填字段为 `title`、`date`、`category` 和 `summary`。`category`
必须是四个固定分类之一。文章输出文件名由 Markdown 文件名决定，
例如 `content/java/concurrency.md` 生成
`articles/java/concurrency.html`。

## 页面设计

整体采用经典双栏布局和偏纸张质感的视觉风格：

- 暖白背景、深灰正文、低饱和强调色。
- 顶部显示博客名称和四个分类菜单。
- 首页主栏展示最新文章，侧栏展示分类、文章数量和个人简介。
- 窄屏设备改为单栏，分类和个人信息移到文章列表之后。
- 排版同时照顾长篇文字与代码阅读。

文章页包含：

- Markdown 渲染后的标题、段落、列表、引用、图片和表格。
- 语法高亮后的代码块。
- 根据正文标题自动生成的文章目录。
- 上一篇和下一篇文章导航。
- 返回顶部操作。

分类页展示分类名称、简介和该分类下按日期倒序排列的文章。没有文章的
分类仍生成页面，并明确显示“暂无文章”。

## 构建流程

执行 `npm run build` 时：

1. 递归扫描 `content/` 下的 Markdown 文件。
2. 解析并校验文章元数据。
3. 将 Markdown 正文转换为 HTML，并生成标题锚点与文章目录。
4. 按日期倒序排列文章，计算分类文章数量。
5. 为每篇文章计算同一分类内的上一篇和下一篇。
6. 生成首页、四个分类页和全部文章页。
7. 写入所有页面需要的本地样式、脚本和代码高亮资源。

构建过程不依赖远程 CDN，生成后的博客离线也能完整浏览。

## 错误处理

构建采用明确失败策略。发现以下问题时停止构建、返回非零退出码，并在
错误信息中包含源文件路径和原因：

- 缺少必填元数据。
- 日期不是合法的 `YYYY-MM-DD`。
- 分类不属于固定分类。
- 两篇文章生成相同输出路径。
- Markdown 无法解析或输出目录无法写入。

空分类不是错误，正常生成空状态页面。

## 测试与验收

自动化测试覆盖：

- 元数据解析和必填字段校验。
- 分类校验与日期排序。
- Markdown 标题锚点和目录生成。
- 首页、分类页、文章页的相对链接。
- 上一篇和下一篇的边界情况。
- 从示例内容执行一次完整构建。

浏览器验收覆盖：

- 直接双击根目录 `index.html` 可以进入博客。
- 首页菜单、分类页和文章页之间可以完整跳转。
- 文章显示为网页排版，不暴露 Markdown 原文。
- 桌面双栏和移动单栏布局均无横向溢出。
- 代码块、目录、上一篇/下一篇和返回顶部可正常使用。

## 首版成功标准

作者可以新增或修改 Markdown，执行一次 `npm run build`，随后直接双击
`index.html` 浏览更新后的博客；四个分类、最新文章和全部文章页面均能
在无服务器、无网络的环境中正常阅读。
