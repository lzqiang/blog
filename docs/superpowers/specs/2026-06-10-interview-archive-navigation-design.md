# 面试题年月归档导航设计

## 目标

在顶部导航的“面试题”分类下增加按年份、月份组织的归档目录。作者继续
通过 Markdown front matter 中的 `date` 字段管理文章日期，构建程序自动
生成目录和归档页，不需要手工维护年份或月份。

## 范围

本次仅为“面试题”分类增加年月归档。Java、人工智能、文学创作和个人随笔
保持现有导航与分类页行为。

归档目录只显示已有文章的年份和月份，不显示空年份或空月份。年份和月份
均按时间倒序排列。

## 导航交互

顶部“面试题”导航拆分为两个操作：

- 点击“面试题”文字进入现有的全部面试题列表页。
- 点击文字旁的箭头打开或关闭归档面板。

归档面板先展示年份，每个年份下展示该年存在文章的月份。点击年份进入
年度归档页，点击月份进入月度归档页。点击导航以外的区域、按下 Escape，
或选择一个归档链接后关闭面板。

箭头按钮使用原生 `button`，包含可读的无障碍名称，并通过
`aria-expanded` 表示面板状态。归档面板与按钮通过 `aria-controls`
关联。键盘用户可以使用 Tab 依次访问年份和月份链接。

桌面端面板显示在“面试题”导航项下方。移动端沿用点击展开行为，面板进入
正常文档流，年份和月份纵向排列，避免横向溢出。

## 数据模型

站点模型从“面试题”分类的文章中提取 `date`：

1. 按四位年份分组。
2. 在每个年份内按两位月份分组。
3. 年份和月份均倒序排列。
4. 每个分组保存对应的文章列表，文章继续按日期倒序排列。

日期来源仍是现有 Markdown front matter，不新增元数据字段。不存在面试题
文章时，不显示箭头和归档面板，现有空分类页保持可访问。

## 页面与路径

保留现有全部面试题页面：

```text
categories/interview.html
```

新增年度和月度静态页面：

```text
categories/interview/2026.html
categories/interview/2026/06.html
```

年度页标题使用“2026 年面试题”，列出该年全部面试题。月度页标题使用
“2026 年 06 月面试题”，列出该月全部面试题。两个页面复用现有分类页的
文章列表样式与空状态组件，但构建程序只为非空分组生成页面。

所有链接继续通过现有相对路径工具生成，保证从 `file://` 打开构建结果时
仍可跳转。

## 实现边界

归档分组属于站点模型，模板只负责渲染已经排序好的数据。布局模板仅对
`interview` 分类输出归档导航，构建模块负责写入年度和月度页面。

前端脚本只管理面板开关、点击外部关闭和 Escape 关闭，不负责计算目录。
因此关闭 JavaScript 时，“面试题”主分类链接仍然可用，生成内容也不会
依赖浏览器运行时处理。

## 测试与验收

自动化测试覆盖：

- 多篇面试题按年份和月份正确分组、倒序排列。
- 不生成没有文章的年份和月份。
- 非面试题文章不进入该归档。
- 导航包含主分类链接、箭头按钮和无障碍属性。
- 年度页和月度页使用正确的相对资源及文章链接。
- 完整构建生成预期的年度、月度 HTML 文件。
- 面试题为空时不显示归档开关。
- 前端脚本支持按钮切换、点击外部关闭和 Escape 关闭。

浏览器验收覆盖：

- 桌面端点击箭头后，面板在导航项下方显示且不遮挡关键内容。
- 点击“面试题”、年份和月份分别进入正确列表页。
- 移动端目录纵向排列且没有横向滚动。
- 键盘可以打开、访问并关闭目录。
- 直接通过 `file://` 浏览时全部归档链接有效。

## 成功标准

新增一篇带有新年份或新月份日期的面试题 Markdown 后，执行
`npm run build`，导航目录及对应归档页自动出现；删除该分组最后一篇文章
并重新构建后，空的年份或月份目录及页面自动消失。
